# S18 — LLM 提示词优化 + 联网检索（新闻/公告/舆情）

> 承接 S17（外部因子扩展）。解决两个问题：①因子数据注入后提示词方向偏"粗暴"
> （裸字段+裸数字平铺，无单位/口径/优先级/分析框架）；②让 LLM 具备补齐
> 新闻/公告/舆情的联网能力（S17 判定这些维度端上无稳定公开源）。不破坏既有
> 业务链路，全部改动向后兼容（新增参数均带默认值，既有调用点/单测零影响）。

## 一、提示词渲染优化（纯提示词工程）

- **新增 `QA/PromptFieldFormatting.swift`**：把 ContextPack 的裸 item 渲染成
  带单位/口径/中文标签的行，并按短线相关性排序（资金/情绪/关键价先，慢变量后）。
  - 金额自动 元/万元/亿元 + 正负号（`主力净流入：+1.24亿元`）。
  - 百分比/量比/评分/布尔/价格数组各有专门格式；`range_position` 按 0–1 折算成 %。
  - 未知字段回退原样输出，绝不丢字段。
- **`PromptBuilder` 系统提示词升级**为**短线分析师人设 + 三维分析框架**
  （资金面优先 → 情绪面校验 → 技术面定位 → 三维交叉取舍），并强制区分
  "规则数据"与"模型推断"。保留原有全部硬性规则（不得编造、数据快照时间、
  数据质量等级、结构化买卖点 ideal_buy/secondary_buy/stop_loss/take_profit、
  levels 依据、免责声明）——既有单测断言全部仍成立。
- `PromptBuilder.build` 新增 `options: Options`（`webSearchEnabled`），默认关闭。

## 二、联网检索能力（两条路，用户按需选）

现状核实：`OpenAICompatibleLLMService` 原本只发 `{model, messages}`，**无 tools、
无联网**——能否检索完全取决于所选 model 是否自带搜索。故提供两条路：

### 路 A：搜索型模型（零额外 Key）
- `LLMConfigurationStore` 新增 `presets`（含 **Perplexity `sonar`/`sonar-pro`** 等
  自带联网模型）与 `webSearchEnabled` 开关（持久化）。
- 设置页可一键套用预设；带地球图标的模型自带联网。开启后系统提示词切换为
  "允许联网检索最新新闻/公告/舆情，须注明来源与时间，且不得覆盖本地数据"。

### 路 B：web_search 工具轮次（模型不自带搜索时）
- **新增 `QA/WebSearchService.swift`**：`WebSearchService` 协议 + `TavilyWebSearchService`
  （`api.tavily.com/search`，单 Key）。可插拔，可 mock 单测。
- **新增 `QA/WebSearchAPIKeyStore.swift`**：Tavily Key 存 Keychain（与 LLM Key 独立）。
- **`OpenAICompatibleLLMService` 扩展**：`Configuration.webSearch` 可选；存在时
  `generate` 走 function-calling 工具轮次（模型请求 `web_search` → 端上执行 →
  回填 → 再答，最多 N 轮，超限强制作答）。单源检索失败不致命，回填错误说明
  让模型基于本地数据兜底作答。`webSearch == nil` 时行为与原来完全一致。
- `makeService`：仅当"开启联网 **且** 存了 Tavily Key"才挂工具轮次；开启但无 Key
  时视为模型自带搜索（路 A），只放开提示词、不挂工具。

## 连线与 UI

- `WorkspaceChatService.ask/streamAsk` 新增 `options` 参数（默认空）。
- `ChatView`：联网开启时走**非流式** `ask`（工具轮次需完整往返，不适合逐 token 流）；
  关闭时保持原流式体验。错误路径下仍保存已记录的用户提问（不丢问题）。
- 设置页：模型预设列表 + 联网开关 + Tavily Key 输入/删除。

- `Tests/QA/WebSearchToolRoundTests`：Tavily 解析、`firstMessage`/`toolCallQuery`/

- 路 B：填 Tavily Key + 普通模型（如 gpt-4o），确认工具轮次能触发搜索并回填。
